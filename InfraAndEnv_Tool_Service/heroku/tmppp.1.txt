さて、Heroku を使う準備ができたので、今度はローカル開発環境で Rails アプリケーションを作って、Heroku に送る準備をしていきましょう。
0:11
まずは「rails new」でアプリケーションを作っていきます。
0:15
今回はメモ帳アプリのようなものを想定しておきましょう。
0:18
PostgreSQL を Heroku では標準で使うので、最初から PostgreSQL を使うような設定でアプリケーションを作っていきます。
0:27
bundle install などでちょっと時間がかかるときもありますが、出来たらそのフォルダに移動して、「vi Gemfile」で Gemfile をいじっていきます。
0:39
Gemfile の方で therubyracer を有効化しておくと、rails server を動かしたときにエラーが出ないかと思います。
0:48
「rails s」として、動作確認をしていきましょう。
0:52
今回は 192.168.78.90:3000 で立ち上がっているはずなので、ブラウザでその URL を開きます。
0:58
うまく Rails アプリケーションが立ち上がっています。
1:01
ここまでできたので、アプリっぽくするために Scaffold 機能を使っていろいろ設定していきましょう。
1:07
「rails g scaffold Memo title:string body:text」で OK かな…。
1:21
これができたら、あとは「rake db:migrate」としてマイグレーションを実行します。
1:29
そうするとデータベースが作られて、「rails s」とすると立ち上がるはずなので…、ブラウザで見ていきましょう。
1:38
192.168.78.90:3000/memos にアクセスすると CRUD アプリが立ち上がっているはずです。
1:46
適当にデータを登録しておきましょう。
1:50
ここまでできたらアプリケーションができたので、これを git に登録しておきます。
1:57
git に登録することによって簡単に Heroku に登録できるので、やっておきましょう。
2:04
まずは「git init」で Git リポジトリとして初期化します。
2:08
「git add .」として、「git commit -m "initial commit"」としましょう。
2:18
ここまで出来たので、このアプリケーションを git を使って Heroku に送りたいのですが…、ちょっと長くなりそうなので続きは次回にしていきます。
